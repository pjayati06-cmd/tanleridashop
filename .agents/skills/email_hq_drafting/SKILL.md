# Email HQ Drafting Skill

Use this skill when drafting new emails, replying to threads, or reviewing tone and formality in messages sent on behalf of Jayti.

## 1. Core Workflow
1.  **Thread Check**: Always check if a thread with the recipient already exists. If so, reply in that thread.
2.  **Formality Check**: Identify the recipient type:
    *   *Institutional/Official* (Banks, services, complaints)
    *   *Professional Contact*
    *   *Personal/Family*
3.  **Voice Review**: Apply voice-principles.md guidelines.
4.  **Editorial Check**: Apply the structural and formatting conventions below before final approval.

## 2. Formatting & Structure Conventions

*   **Rhythm**: Warm opener (1 sentence) → Context (1-2 sentences) → Ask/Information (1-2 sentences) → Close.
*   **Prose Preference**: Write in prose rather than bullet points for almost all email bodies.
*   **Single Ask**: Limit each email to a single clear request or action item. If there are multiple points, sequence them in separate short paragraphs.
*   **No Clichés**: Never start with "I hope this message finds you well". Start directly with warmth openers like "Hope you're well" or "Hope you're keeping well".
*   **Sign-Off**: Use "Best regards" followed by a line break and "Jayti Pargal."

## 3. Formality Levels by Recipient Type

### Institutional / Customer Care / Complaints
*   **Tone**: Strict prose, no bullet points. Factual, detailed, and specific (include numbers, dates, costs, expected outcomes).
*   **Action Close**: End with a direct request: "Could you please check this at your end and let me know." No emotional padding.
*   **Greeting**: Plain "Hi," with no name.

### Professional Contacts
*   **Tone**: Warm, direct, assured. Use terms like "Please find attached," "As discussed," or "Looking forward to."
*   **Greeting**: "Hi [Name],"

### Personal / Family
*   **Tone**: Direct, conversational, minimal framing.
*   **Greeting**: "Hi [Name]," or "Hey [Name],"
*   **Sign-off**: Keep it brief but still structured.
