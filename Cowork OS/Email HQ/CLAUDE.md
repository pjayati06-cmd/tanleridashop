## Workflow

1. Before drafting, check if a thread with this recipient already exists. If yes, reply in that thread.
2. Read voice-principles.md from 00_Resources.
3. Identify recipient type (institutional, personal/family, professional contact) and match formality.
4. Draft using the structure: opener → context → ask or information → close.
5. Apply the Editorial Rules below before finalising.

## Editorial Rules

Follow my voice principles in 00_Resources (voice-principles.md).

**Greeting defaults**
- For named contacts (professional or personal): "Hi [Name],"
- For institutional or generic addresses (customer care, no-reply contexts, unnamed recipients): plain "Hi," with no name.

**Sign-off**
- Default sign-off is "Best regards" followed by a line break and "Jayti Pargal." Do not shorten to "Best, Jayti" or use "Thanks" as a closing unless the thread is very informal and personal.

**Formality by recipient type**
- Institutional / complaints (banks, services, official bodies): prose only, no bullets. State facts and figures precisely — specific amounts, dates, what was expected vs. what happened. End with a single, clear action request: "Could you please check this at your end and let me know." No drama, no padding.
- Personal/family: minimal framing. Forwards or quick messages need no lengthy preamble. Sign-off still applies.
- Professional contacts: follow voice-principles.md tone guidance. Warmth openers apply here.

**Structure conventions**
- Prose over bullets in almost all email contexts. Use bullets only if listing genuinely parallel items (e.g., multiple deliverables, a checklist).
- One ask per email. If there are multiple points, sequence them as short paragraphs, not a bulleted list.

*Note: These rules are based on a limited email sample (2 sent emails from this Gmail account in the last 4 weeks). Patterns will be refined as more data is available.*
